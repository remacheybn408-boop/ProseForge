# ProseForge Web 本地化、水墨视觉与设置页设计

## 目标

改善普通用户首次使用 ProseForge 的理解成本和视觉一致性：默认使用简体中文，保留中英文切换；将现有暖色后台式界面调整为水墨书卷风；重组模型设置页，让非技术用户能理解每个字段和操作。

## 范围

- Web SPA 的界面文案、语言选择和语言持久化。
- 全局设计令牌、导航、项目页、工作台、工作流、上下文和设置页视觉样式。
- 设置页的模型连接、写作角色和连接状态分区。
- Web 单元测试、生产构建和 Docker 启动后的手动验收。

不翻译用户创建的项目标题、章节内容、提示词或模型返回内容；不改动 API 数据结构和认证流程；不引入图片资源或大型国际化依赖。

## 语言方案

- 语言字典支持 `zh-CN` 和 `en-US`。
- 默认语言为 `zh-CN`。
- 语言选择位于左侧导航底部，显示“中文 / English”。
- 选择保存到 `localStorage`，刷新和重新登录后保持。
- 界面组件通过统一翻译函数取文案，避免在组件中散落语言判断。
- API 错误若已有可识别错误码，前端提供中英文用户提示；未知错误使用通用提示。

## 视觉方案

- 背景使用宣纸米白，正文使用墨黑，主要行动使用朱砂色。
- 使用 CSS 渐变、噪点感和轻量水墨晕染表现层次，不依赖外部图片。
- 导航保持清晰的高对比和键盘可操作性；选中项使用朱砂印章式强调。
- 编辑器使用书稿式衬线字体、舒适行高和更宽留白。
- 减少大面积卡片、重边框和后台表单感，保留必要的分隔线和状态反馈。
- 响应式布局继续支持窄屏，语言切换和设置表单不得被遮挡。

## 设置页方案

### 模型服务

- 标题使用“模型服务连接”，说明密钥只用于服务端调用且不会回显。
- 字段显示为“服务商”“API 密钥”“接口地址（可选）”。
- 接口地址和高级配置提供简短说明与示例；默认不暴露不必要的底层术语。
- 保存后清空密钥输入框，列表只显示掩码值。

### 写作角色

- 将 Writer / Editor 显示为“写作模型 / 审校模型”。
- 模型 ID 显示为“模型名称 / Model ID”，并说明这是服务商提供的模型标识。
- 保留现有角色和模型配置 API，不改变数据语义。

### 连接状态

- 每个已保存服务显示“已连接”“未连接”或“检查失败”。
- 操作按钮使用“测试连接”，错误信息说明下一步检查方向。

## 验证标准

- 默认首次打开为简体中文。
- 切换 English 后主要导航、项目页和设置页文案切换，刷新后保持 English。
- 切回中文后文案恢复，用户内容不被改写。
- 设置页普通用户可理解字段用途，并能完成保存与测试连接。
- Docker 内 Web 测试全部通过，Vite 生产构建通过。
- Docker 启动后 Web、API、Worker、Scheduler、PostgreSQL、Redis 健康。
